# Summary Report: Speaker Session – Cloud Journey, Scalable Architecture & DevOps

**Date & Time:** 9:00 – 12:00, June 13, 2026

One half-day event with **four consecutive sessions** (Session 1 → Session 4) in the same program.

### Event Objectives

- Share real learning and career paths from the First Cloud AI Journey (FCAJ) program toward AWS Partner readiness.
- Present a practical, scalable URL shortening service built on AWS.
- Introduce enterprise culture and professional mindset in cloud/tech organizations.
- Provide orientation on the DevOps Engineer role, skills, and day-to-day responsibilities.

### Speakers (by session)

- **Session 1 – Danh Hoang Hieu Nghi:** *From First Cloud AI Journey to AWS Partner*.
- **Session 2 – Dinh Trung Kien & Nguyen Minh Tho:** *Scalable URL shortening service on AWS* (shared presentation).
- **Session 3 – Mr. Cuong Nguyen:** *Enterprise culture*.
- **Session 4 – Truong Hoang Trong:** *DevOps Engineer*.

### Key Highlights

Four sessions within the same event (12/06/2026, 9:00 – 12:00):

#### Session 1 – From First Cloud AI Journey to AWS Partner (Danh Hoang Hieu Nghi)

- Overview of the FCAJ learning path and how structured cloud training builds toward partner-level competency.
- Milestones from beginner labs to production-minded architecture and customer-facing skills.
- How certifications, projects, and community involvement support the journey to AWS Partner ecosystem roles.

#### Session 2 – Scalable URL shortening service on AWS (Dinh Trung Kien & Nguyen Minh Tho)

- Problem statement and requirements for a URL shortener that can scale with traffic growth.
- AWS services typically involved: API layer, durable storage, caching, and security at the edge.
- Design considerations for high availability, low latency, and cost-aware scaling on AWS.

#### Session 3 – Enterprise culture (Mr. Cuong Nguyen)

- Values and behaviors expected in professional cloud and software teams.
- Communication, ownership, and collaboration norms in enterprise environments.
- How culture supports delivery quality, learning, and long-term career growth.

#### Session 4 – DevOps Engineer (Truong Hoang Trong)

- Core responsibilities of a DevOps Engineer: CI/CD, automation, monitoring, and reliability.
- Relationship between development, operations, and cloud infrastructure in modern teams.
- Skills to develop during the bootcamp that align with a DevOps career path (IaC, observability, pipelines).

### Key Takeaways

#### Cloud journey & partnership readiness

- FCAJ is a stepping stone; consistent project work and certification planning accelerate partner-track readiness.
- Technical depth must be paired with communication and solution-oriented thinking.

#### Scalable architecture on AWS

- Even common applications (URL shorteners) illustrate important patterns: decoupling, caching, and managed services.
- Architecture choices should balance performance, cost, and operational simplicity.

#### Professional growth

- Enterprise culture emphasizes accountability, teamwork, and continuous improvement—not only individual coding skill.
- DevOps bridges build and run concerns; automation and observability are central to the role.

### Event Experience

Attending this **single half-day event** (four sessions back to back) complemented weekly workshops with **industry and peer perspectives**. Highlights included:

#### Sessions 1–4 in one program

- **Session 1 – Danh Hoang Hieu Nghi:** connected bootcamp learning to a longer cloud career and AWS Partner direction.
- **Session 2 – Dinh Trung Kien & Nguyen Minh Tho:** showed how a familiar use case maps to real AWS building blocks.
- **Session 3 – Mr. Cuong Nguyen:** added context on working effectively inside professional organizations.
- **Session 4 – Truong Hoang Trong:** clarified what DevOps work looks like in practice and which skills to prioritize.

#### Community and motivation

- Four sessions in one morning reinforced that cloud careers combine technical depth, culture, and delivery discipline.
- Q&A and discussion helped relate speaker topics to internship project work.

#### Lessons learned

- Scalable design starts with clear requirements and service boundaries, not only choosing popular AWS products.
- DevOps and enterprise culture both reward consistency, automation, and clear communication across teams.
